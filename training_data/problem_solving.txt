Why does my loop run one extra time when using <= instead of <?|Using <= includes the boundary value, so a loop from 0 to n with <= runs n+1 times instead of n. Change to < if you want exactly n iterations.
A function returns None instead of the expected value. What's the most likely cause?|The function is missing a return statement on one of its code paths. Check all branches, especially inside if/else blocks, to ensure every path returns a value.
How do you decide between using a list and a dictionary in Python?|Use a list when you need ordered elements accessed by index. Use a dictionary when you need fast key-based lookup. If you're searching a list repeatedly by a property, a dict is almost always better.
My recursive function causes a stack overflow. How do I fix it?|Check your base case — it's either missing, unreachable, or the recursive call doesn't reduce toward it. Add or fix the base case, or convert to an iterative approach with an explicit stack.
Two threads updating the same counter give wrong results. Why?|Read-modify-write on a shared variable is not atomic. Both threads can read the same value before either writes back. Use a lock, atomic operation, or thread-safe counter.
When should I use a SQL JOIN vs a subquery?|Use a JOIN when you need columns from both tables in the result. Use a subquery when you need to filter based on an aggregate or existence check. JOINs are usually faster for large datasets.
My API returns 200 but the client shows an error. How do I debug?|Check the response body — the API might return 200 with an error payload. Also check CORS headers, content-type mismatches, and whether the client is parsing the response correctly.
How do you find a bug that only happens in production?|Add structured logging around the suspected area, reproduce with production-like data locally, check environment differences (config, versions, data volume), and use feature flags to isolate the change that introduced it.
A sorted array search is slow. What data structure should I use instead?|If the array is static, use binary search for O(log n). If it changes frequently, use a balanced BST or skip list. For exact-match lookups, a hash map gives O(1) average.
My database query is slow but returns correct results. First step?|Run EXPLAIN on the query to see the execution plan. Look for full table scans, missing indexes, or unnecessary joins. Add indexes on columns used in WHERE and JOIN clauses.
How do you handle a function that needs to return both a result and an error?|In Go, return a tuple (result, error). In Python, raise exceptions for errors. In Rust, use Result<T, E>. In JavaScript, use try/catch or return an object with success/error fields.
My CSS layout breaks on mobile. What's the systematic approach?|Check viewport meta tag, use browser dev tools responsive mode, inspect element widths for overflow, verify flex/grid properties have proper wrapping, and test with actual device aspect ratios.
When is it better to duplicate code than abstract it?|When the two pieces serve different domains and will likely diverge. Premature abstraction couples unrelated things. Wait until you have three similar cases before extracting a shared function.
A merge conflict spans 200 lines. How do you resolve it safely?|Understand both branches' intent from their commit messages. Use a three-way diff tool. Resolve in small sections. Run tests after resolving. Don't just pick one side blindly.
How do you debug a memory leak in a long-running process?|Take heap snapshots at intervals and compare them. Look for objects that grow without bound. Common causes: event listeners not removed, caches without eviction, closures holding references.
My test passes locally but fails in CI. What do I check?|Environment differences: OS, dependency versions, timezone, locale, file ordering, parallelism, network access, and test isolation. CI may run tests in parallel or in a different order.
Should I use an ORM or raw SQL?|Use an ORM for standard CRUD operations and rapid development. Use raw SQL for complex queries, performance-critical paths, or when you need database-specific features the ORM doesn't support.
A user reports a bug but can't reproduce it. Next steps?|Gather exact steps, browser/OS, account state, timestamps. Check server logs for errors at that time. Try their exact data. Consider race conditions or state that only exists after specific user flows.
How do you decide the right level of error handling?|Validate at system boundaries (user input, external APIs, file I/O). Trust internal function calls between your own code. Don't catch exceptions you can't meaningfully handle — let them propagate.
My async function completes but the callback never fires. Why?|The promise or future might be swallowed — ensure you're awaiting it or attaching .then(). Also check if an unhandled error is silently caught, or if the event loop is blocked.
When should you denormalize a database?|When read performance matters more than write consistency, you have heavy JOIN queries slowing down reads, or you're building a read-optimized view. Always keep a normalized source of truth.
How do you debug an off-by-one error?|Trace through the loop manually with the boundary values (empty input, single element, exactly-N elements). Check whether indices are 0-based or 1-based, and whether bounds are inclusive or exclusive.
My Docker container starts but the app inside crashes. How to debug?|Run the container interactively with docker run -it --entrypoint /bin/sh. Check logs with docker logs. Verify environment variables, mounted volumes, and that dependencies are installed in the image.
How do you decide between polling and webhooks?|Use webhooks when the service supports them and you need real-time updates. Use polling when webhooks aren't available, you need simplicity, or the data changes infrequently enough that periodic checks suffice.
A regex matches too much text. What's wrong?|Greedy quantifiers (.* or .+) match as much as possible. Use lazy quantifiers (.*? or .+?) or be more specific about what characters to match instead of using dot.
My git rebase has conflicts on every commit. Better approach?|If the branch is far from main, consider squashing your branch into one commit first, then rebasing. Or merge main into your branch instead. Rebase is painful when branches have diverged significantly.
How do you trace a bug through a microservices system?|Use distributed tracing (correlation IDs passed through headers). Check logs for the request ID across services. Start from the user-facing error and follow the trace backward through the call chain.
When should you use an index vs a full table scan?|Use an index when selecting a small percentage of rows (<10-15%). A full scan is faster when reading most of the table, since random I/O from index lookups is slower than sequential reads.
My Python script works in 3.9 but fails in 3.11. How to diagnose?|Check the changelog for deprecated/removed features. Common issues: match statement syntax, dictionary ordering guarantees, removed modules, or changed default behaviors. Use pyenv to test both versions.
How do you choose between REST and GraphQL?|Use REST for simple CRUD APIs with predictable data shapes. Use GraphQL when clients need flexible queries, you have multiple consumers needing different data subsets, or you want to reduce over-fetching.
A function works correctly but is too slow. First optimization step?|Profile it. Don't guess. Use a profiler to find the actual bottleneck. Common fixes: reduce algorithmic complexity, cache repeated computations, batch I/O operations, or eliminate unnecessary allocations.
My application hangs with no errors. How do I find the cause?|Attach a debugger and inspect thread stacks, or send SIGQUIT to get a thread dump. Look for deadlocks (two threads waiting for each other's locks) or infinite loops with no I/O.
How do you decide when to refactor vs rewrite?|Refactor when the structure is salvageable and you understand the code. Rewrite when the codebase fights every change, has no tests, and the domain is well-understood enough to rebuild correctly.
My unit test is testing implementation details. How do I fix it?|Test behavior, not internals. Assert on outputs for given inputs, observable side effects, or public API contracts. Don't assert on private method calls, internal state, or execution order.
A database migration fails halfway. What's the recovery plan?|If the migration isn't in a transaction, you need to manually reverse completed steps. Write down migrations as idempotent operations. Always test migrations on a copy of production data first.
How do you debug a CORS error?|Check the server's Access-Control-Allow-Origin header matches the requesting origin. Verify allowed methods and headers. Ensure preflight OPTIONS requests are handled. The error is always on the server config, not the client.
When should you use a message queue vs direct API calls?|Use a queue when the consumer is slower than the producer, you need retry/dead-letter handling, operations can be async, or you want to decouple services. Direct calls for synchronous request-response patterns.
My hash map lookup returns null but the key exists. Why?|The key object's hashCode or equals method is inconsistent — it hashes to a different bucket than when it was inserted. Ensure hashCode and equals use the same fields and are immutable after insertion.
How do you handle a third-party API that's unreliable?|Add retries with exponential backoff, circuit breakers to stop hammering a failing service, timeouts on every call, fallback responses or cached data, and alerts when the circuit opens.
My program works with small inputs but crashes on large ones. Likely cause?|Memory — you're loading everything into memory at once. Stream or batch the data. Also check for O(n²) or worse algorithms that become impractical at scale.
How do you decide between a monolith and microservices?|Start with a monolith. Split into services only when you have clear domain boundaries, independent scaling needs, or separate team ownership. Microservices add network complexity, deployment complexity, and debugging difficulty.
A null pointer exception occurs deep in a call chain. How to trace it?|Read the stack trace bottom-up to find where null was introduced. Add null checks or assertions at the entry point of the function that received null. Check if an upstream function has a missing return or failed lookup.
How do you safely deploy a database schema change with zero downtime?|Use expand-and-contract: add the new column/table, deploy code that writes to both, backfill old data, deploy code that reads from new, then drop the old column. Never rename or remove in one step.
My event listener fires multiple times. What's happening?|The listener is being registered multiple times — likely in a render loop, useEffect without cleanup, or repeated initialization. Ensure you remove the old listener before adding a new one.
When should you cache data and when shouldn't you?|Cache when data is read far more often than written, expensive to compute, and tolerant of staleness. Don't cache when data changes frequently, correctness requires freshness, or cache invalidation is too complex.
How do you debug a flaky test?|Run it in isolation, then with the full suite. Check for shared mutable state, time-dependent logic, random ordering, network calls, or file system dependencies. Add deterministic seeds and mock external deps.
My API response time doubled after a deploy. How to investigate?|Compare the deploy diff against performance profiles. Check for N+1 queries, missing database indexes on new queries, added middleware, increased payload sizes, or new synchronous external calls.
How do you decide whether to throw an exception or return an error code?|Throw exceptions for truly exceptional, unexpected situations. Return error codes or Result types for expected failure modes that callers should handle. Exceptions shouldn't be used for flow control.
A background job processes the same item twice. How to prevent it?|Make the job idempotent — processing the same item twice should have the same effect as once. Use unique constraints, check-then-act with locks, or deduplication keys in your queue.
My web page loads slowly. How do I systematically diagnose it?|Open browser DevTools Network tab. Check largest requests, blocking resources, waterfall pattern. Then check Performance tab for long tasks. Common fixes: compress assets, lazy-load images, reduce JavaScript bundles.
How do you choose between optimistic and pessimistic locking?|Use optimistic locking (version checks) when conflicts are rare — most reads, few concurrent writes. Use pessimistic locking (SELECT FOR UPDATE) when conflicts are frequent and the cost of retry is high.
A function returns stale data after an update. Likely cause?|Caching — either application-level, ORM-level, or CDN. Check if the function reads from a cache that wasn't invalidated after the write. Also check for read replicas with replication lag.
How do you debug a segmentation fault in C?|Compile with -g and run under gdb or valgrind. The segfault usually means dereferencing a null or freed pointer, buffer overflow, or stack overflow. Valgrind shows exact location and cause.
When should you use inheritance vs composition?|Prefer composition. Use inheritance only for genuine is-a relationships where you need polymorphism. Composition is more flexible, avoids tight coupling, and doesn't break when the parent class changes.
My distributed system has inconsistent data across nodes. How to fix?|Identify your consistency requirement. Use consensus protocols (Raft, Paxos) for strong consistency. For eventual consistency, add vector clocks or CRDTs. Always reconcile conflicts with a defined strategy.
How do you debug a program that produces correct output but wrong exit code?|Check explicit exit/sys.exit calls, uncaught exceptions in finally blocks, signal handlers, and subprocess calls whose exit codes propagate. Also check if a test framework is overriding the exit code.
My React component re-renders too often. Diagnosis steps?|Use React DevTools Profiler to see what triggers re-renders. Common causes: new object/array references created each render, missing useMemo/useCallback, context changes, or parent re-rendering unnecessarily.
How do you decide between synchronous and asynchronous processing?|Use sync when the caller needs the result immediately and latency is acceptable. Use async when the operation is slow, the result isn't needed immediately, or you need to handle load spikes gracefully.
A SQL query returns duplicate rows unexpectedly. Why?|A JOIN is matching multiple rows — likely a one-to-many relationship you forgot about. Use DISTINCT, fix the JOIN condition, or aggregate the results. Check your data model for the actual cardinality.
How do you approach debugging a race condition?|Add logging with timestamps and thread IDs around the suspected section. Use thread sanitizers (TSan). Try to make the race deterministic by adding sleep to widen the window. Then fix with proper synchronization.
My Python import works in script but fails in package. Why?|Relative vs absolute import issue. Inside a package, use relative imports (from . import module) or fully qualified absolute imports. Running a module directly vs as part of a package changes the import resolution.
When should you use a database transaction?|When multiple operations must succeed or fail together — transferring money, creating related records, or any sequence where partial completion leaves data in an invalid state.
How do you debug a 502 Bad Gateway error?|The upstream server is down or timing out. Check if the application process is running, check its logs for crashes, verify the reverse proxy config points to the right port, and check if the app is overloaded.
A binary search implementation doesn't find existing elements. Common bug?|Off-by-one in the mid calculation or boundary updates. Ensure mid = low + (high - low) / 2 to avoid overflow, and that low and high converge correctly without skipping elements or looping infinitely.
How do you decide between PUT and PATCH in a REST API?|PUT replaces the entire resource — client sends the full object. PATCH updates specific fields — client sends only what changed. Use PUT for simple resources, PATCH when partial updates are common.
My application works but leaks file descriptors. How to find and fix?|Use lsof to count open descriptors per process. Check for files, sockets, or database connections opened but never closed. In Python use with statements, in Java use try-with-resources, to ensure cleanup.
How do you handle a schema change when multiple services share a database?|Don't share databases between services. If you must, use backward-compatible changes only (add columns, don't remove or rename). Better: give each service its own schema and use events or APIs to sync.
My floating point comparison fails for values that should be equal. Fix?|Never compare floats with ==. Use an epsilon: abs(a - b) < 1e-9. Or use decimal types for exact arithmetic. The issue is that 0.1 + 0.2 != 0.3 in IEEE 754 floating point.
How do you debug a Kubernetes pod stuck in CrashLoopBackOff?|Run kubectl logs to see the crash reason. kubectl describe pod shows events and exit codes. Common causes: missing config/secrets, failed health checks, OOM kills, or application startup errors.
When should you use a read-write lock vs a mutex?|Use a read-write lock when reads vastly outnumber writes and the read operation is non-trivial. If reads and writes are balanced, or the critical section is short, a simple mutex has less overhead.
My GraphQL query works in the playground but fails from the client. Why?|Check that the client sends the query as a POST with Content-Type: application/json. Verify the query string is properly escaped. Check authentication headers and CORS settings between client and server.
How do you decide between eager and lazy loading in an ORM?|Use eager loading when you know you'll need the related data (avoids N+1 queries). Use lazy loading for optional relationships accessed infrequently. Profile to see which pattern generates fewer total queries.
A deployment succeeds but the new version isn't serving traffic. What's wrong?|Health checks might be failing — the new version starts but doesn't pass readiness probes. Check the health endpoint, verify required environment variables and connections, and inspect the load balancer target group status.
How do you debug a function that produces different results on different runs?|Look for sources of non-determinism: uninitialized variables, dictionary/set ordering, floating point on different hardware, system time, random without seed, or concurrent access to shared state.
When should you use WebSockets vs Server-Sent Events vs polling?|WebSockets for bidirectional real-time communication. SSE for server-to-client push (simpler, auto-reconnect, works through proxies). Polling for low-frequency updates where simplicity matters most.
My git branch has commits I didn't make. What happened?|You likely merged or rebased from another branch that included those commits. Use git log --oneline --graph to see the merge structure. Use git reflog to trace how your branch pointer moved.
How do you approach a codebase you've never seen before?|Read the README and entry point. Run the app. Trace one user-facing feature end to end through the code. Don't try to understand everything — understand one slice deeply, then expand.
A try-catch block silently swallows errors. How do you fix this anti-pattern?|At minimum, log the error with stack trace. Better: only catch specific exceptions you can handle, and re-raise or propagate everything else. Never use a bare except or catch(Exception) without re-throwing.
How do you decide between an in-memory cache and a distributed cache?|In-memory (dict, LRU) for single-process apps where cache loss on restart is acceptable. Distributed (Redis, Memcached) when multiple instances need shared cache, or cache must survive restarts.
My CI pipeline takes 45 minutes. How do I speed it up?|Profile the pipeline to find the slowest steps. Common fixes: parallelize test suites, cache dependencies, use smaller Docker base images, run only affected tests on incremental changes, split into stages.
How do you debug an infinite loop that doesn't crash?|Add a counter that throws after N iterations. Or attach a debugger and break into the running process to see where it's stuck. Check loop exit conditions — often a condition that's always true or a variable that never updates.
A webhook endpoint processes events out of order. How to handle this?|Include a sequence number or timestamp in events. Store the last processed sequence and skip or reorder events accordingly. Design handlers to be idempotent so reprocessing is safe.
